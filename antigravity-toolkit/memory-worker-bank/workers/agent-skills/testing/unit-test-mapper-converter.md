This Agent Skill provides comprehensive guidance and patterns for unit testing your mapper and converter components, critical for ensuring data integrity and consistency. Whether you're leveraging the power of MapStruct for automated mapping or crafting custom transformation logic, mastering these testing techniques will significantly reduce bugs related to incorrect data representation. # When to Use This Skill
- •Validating a new MapStruct mapper for a complex DTO-entity pair with multiple nested objects.
- •Ensuring a custom converter correctly handles null values and default assignments during data transformation.
- •Refactoring existing mapping logic and needing to confirm all transformations remain accurate.
- •Implementing a new feature that requires converting date formats or custom types between layers.

# Pro Tips
- 💡Utilize data builders or factories to easily create test data, improving readability and reducing boilerplate in your test cases.
- 💡Focus on edge cases: thoroughly test null inputs, empty collections, boundary values, and invalid data types to ensure mapper robustness.
- 💡Leverage assertion libraries like AssertJ for fluent and expressive assertions, making your test failures clearer and easier to debug.
