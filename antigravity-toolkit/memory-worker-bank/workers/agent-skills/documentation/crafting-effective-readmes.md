Effective READMEs are the cornerstone of any successful project, whether for open-source contributors or future-you revisiting a forgotten script. This AI Agent Skill elevates your documentation game by providing structured guidance on tailoring content to your specific audience and project type. Forget generic templates; this tool helps you identify crucial information, streamline your writing process, and ensure your READMEs are always clear, comprehensive, and actionable. From initial creation to ongoing updates, make your projects more accessible and maintainable with expertly crafted documentation.

# When to Use This Skill
- •Starting a new software project and needing a comprehensive, audience-specific README from scratch.
- •Adding new features or modules to an existing project and requiring an update to the documentation.
- •Reviewing an outdated or unclear README to identify gaps and propose improvements.
- •Documenting a configuration folder or a small script for personal future reference.

# Pro Tips
- 💡Before using, clearly define your primary audience and their technical proficiency. This skill excels when you provide this context upfront.
- 💡Utilize the 'Project Types' guidance within the skill to select the most appropriate template, even if your project doesn't fit perfectly into a single category.
- 💡After generating content, always review it manually for tone, flow, and specific project details that only you can provide. The AI provides a strong framework, but human touch is essential.
