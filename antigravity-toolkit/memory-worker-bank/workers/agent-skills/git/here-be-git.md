This skill automates the crucial first step for any development project: setting up version control with Git. Beyond just `git init`, it thoughtfully integrates AI agent workflow best practices by adding commit instructions to your `AGENTS.md` and `CLAUDE.md` files. This ensures your coding assistant is primed for consistent and descriptive commit habits from the outset. By streamlining this foundational process, you save time and establish a disciplined approach to project management, allowing you to focus immediately on coding rather than configuration.

# When to Use This Skill
- •Starting a new coding project from scratch in an empty directory.
- •Bringing an unversioned existing codebase under Git control.
- •Setting up a new project where you want your AI assistant to follow specific Git commit guidelines.
- •Quickly preparing a demo environment that needs version control enabled.

# Pro Tips
- 💡**Customize AGENTS.md**: After initialization, review and further customize the `AGENTS.md` file to include additional specific guidelines pertinent to your team or project's Git workflow, ensuring tailored agent behavior.
- 💡**Integrate .gitignore**: Although not explicitly part of this skill's core function, always follow up by having the agent generate a relevant `.gitignore` file to prevent unwanted files from being committed.
- 💡**Initial Commit**: Instruct the agent to perform an immediate initial commit after the setup is complete, perhaps with a message like 'Initial project setup with Git and agent configs.' to establish a clean starting point.
