Ensuring digital products are accessible to everyone is not just a regulatory requirement but a fundamental principle of inclusive design. This agent skill empowers developers to build interfaces that are usable by individuals with diverse needs, leveraging best practices for WCAG 2.2 compliance. By integrating accessibility from the ground up, you can create more robust, user-friendly applications, expanding your reach and demonstrating a commitment to equitable digital experiences. It’s about building a web that truly works for all, regardless of ability, through thoughtful implementation and continuous improvement.

# When to Use This Skill
- •Auditing an existing web application to identify and remediate WCAG 2.2 Level AA violations and suggest fixes.
- •Developing a new UI component library that inherently supports keyboard navigation, focus management, and ARIA attributes.
- •Integrating mobile-specific accessibility features like iOS VoiceOver or Android TalkBack into a native or hybrid app.
- •Creating accessible forms with proper labeling, error feedback, and semantic structure for screen reader users.

# Pro Tips
- 💡Always test with actual assistive technologies (e.g., NVDA, JAWS, VoiceOver) and real users where possible, as automated tools can only catch a subset of issues.
- 💡Prioritize semantic HTML and native elements before resorting to ARIA, as proper semantics provide a robust and accessible baseline.
- 💡Integrate accessibility checks into your CI/CD pipeline to catch regressions early and maintain compliance throughout the development lifecycle.
