The Brainstorming Agent Skill empowers AI coding assistants to act as a collaborative design partner, guiding users from nascent ideas to structured, actionable specifications. This skill is indispensable for initiating any new feature, component, or system modification, ensuring that user intent, functional requirements, and potential design approaches are thoroughly explored before a single line of code is written. It streamlines the initial conceptualization phase, fostering clarity and preventing costly rework by establishing a robust design foundation through iterative questioning and thoughtful proposal generation.

# When to Use This Skill
- •Initiating a new feature: When you have a high-level idea for a new feature and need to flesh out its details and design.
- •Refactoring an existing component: Before changing an existing module, use this to explore different refactoring strategies and their implications.
- •Designing a new API endpoint: Collaborate to define the endpoint's purpose, request/response structures, and error handling.
- •Planning a complex UI interaction: Work through user flows and component states before implementation to ensure a robust design.

# Pro Tips
- 💡**Be specific with initial prompts:** The clearer your starting idea, the more focused and relevant the agent's questions and design proposals will be.
- 💡**Engage iteratively:** Don't rush the process. Respond thoughtfully to each question and provide feedback on design sections to guide the agent effectively.
- 💡**Use it for discovery, not just validation:** Leverage the agent's ability to propose multiple approaches and trade-offs to uncover solutions you might not have considered.
