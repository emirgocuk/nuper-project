The Subagent-Driven Development skill revolutionizes how AI coding assistants tackle complex implementation plans. By empowering independent subagents to execute discrete tasks, it streamlines development, ensuring each component meets stringent quality and specification standards. This methodology is particularly effective for managing projects that benefit from parallel processing and robust verification steps, fostering rapid iteration and superior code outcomes within a single interactive session. It's a strategic approach for maintaining focus and cohesion across diverse coding challenges.

# When to Use This Skill
- •Executing a multi-step implementation plan where tasks can be tackled independently by separate AI agents.
- •Ensuring high standards of code quality and strict adherence to specifications through automated, two-stage reviews after each task.
- •Managing a software development project that requires modular task completion and verification within the current coding session.
- •Breaking down a large feature or complex problem into manageable, verifiable sub-components for an AI assistant to build iteratively.

# Pro Tips
- 💡Pre-validate Task Independence: Before invoking this skill, thoroughly review your implementation plan to confirm that tasks are genuinely independent to maximize parallelization benefits.
- 💡Define Review Criteria Clearly: Provide explicit guidelines for 'spec compliance' and 'code quality' to guide the subagents and review processes effectively.
- 💡Iterate on Plan Granularity: If subagents frequently struggle or reviews highlight hidden dependencies, consider refining your initial plan to break tasks into even smaller, more atomic units.
