Efficiently steering your coding projects requires a robust system for managing work units. This specialized AI Agent Skill, 'Track Management,' empowers developers to seamlessly navigate the lifecycle of features, bugs, and refactors within the Conductor framework. By providing structured guidance on creating, defining, planning, and completing logical work units—known as tracks—it ensures clarity, fosters team alignment, and optimizes project progress. Leveraging this skill transforms complex development cycles into manageable, traceable processes, enhancing productivity for individual contributors and entire teams using AI-powered coding assistants.

# When to Use This Skill
- •Initiating a new product feature from concept to detailed specification within a Conductor track.
- •Standardizing the planning and execution phases for complex bug fixes, ensuring all steps are documented in `plan.md`.
- •Streamlining refactoring efforts by defining clear scope, rationale, and expected outcomes in a dedicated track.
- •Reviewing and updating existing track documentation (`spec.md`, `plan.md`) to reflect current progress and future changes.

# Pro Tips
- 💡Always begin with a clear, concise problem statement or objective in `spec.md` before detailing the implementation steps in `plan.md` to ensure foundational alignment.
- 💡Regularly update track status markers and metadata within the `tracks.md` registry to maintain an accurate and transparent project overview for all team members.
- 💡Leverage the semantic organization of Conductor tracks for Git-aware operations, making it simpler to revert specific logical units of work or manage branches associated with distinct features or fixes.
