In the rapidly evolving world of AI-driven development, clear communication is paramount. The 'Ask Questions If Underspecified' skill empowers your AI coding assistant to proactively identify and address ambiguities in project requirements. Instead of making assumptions that could lead to rework or incorrect solutions, this skill guides the agent to seek clarification on objectives, scope, constraints, and environment details. By ensuring a shared understanding from the outset, it significantly enhances efficiency, accuracy, and overall project success, fostering a more collaborative and effective AI development workflow.

# When to Use This Skill
- •When a user's request for a new feature lacks details on its exact functionality, acceptance criteria, or target users.
- •If asked to 'optimize the application' without specific performance metrics, areas to focus on, or target environments.
- •When a task involves modifying existing code but doesn't clearly define the scope of changes, affected modules, or potential side effects.
- •Before implementing a data migration, if the old and new schema details, rollback strategy, or impact on existing data are unclear.

# Pro Tips
- 💡Always guide your AI to prioritize questions that prevent the most critical 'wrong work' first, focusing on objective and safety before minor details.
- 💡When the user is unresponsive, prompt the AI to state its assumptions and proposed path forward clearly, allowing for explicit approval or correction.
- 💡Encourage the agent to offer multiple plausible interpretations to the user, making it easier for them to identify and clarify the intended path.
