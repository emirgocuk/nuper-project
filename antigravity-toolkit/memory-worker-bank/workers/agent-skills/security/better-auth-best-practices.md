This skill empowers AI assistants to guide developers through integrating Better Auth, a robust TypeScript-first authentication framework. It goes beyond basic setup, offering insights into secure configuration, environment variable management, CLI usage, and leveraging its modular plugin architecture. This resource helps ensure a robust and compliant authentication system, enabling faster, more secure development workflows by providing instant access to critical integration knowledge and best practices directly within your coding environment. Developers can quickly resolve common setup issues and confidently implement various authentication strategies.

# When to Use This Skill
- •Setting up Better Auth in a new TypeScript project with various authentication methods.
- •Configuring specific Better Auth plugins like OAuth providers, magic links, or passkeys.
- •Troubleshooting environment variable setup or CLI commands for Better Auth migrations and schema generation.
- •Ensuring secure and best-practice configuration for Better Auth secrets and base URLs.

# Pro Tips
- 💡Always generate a strong, minimum 32-character `BETTER_AUTH_SECRET` using secure methods like `openssl rand -base64 32` and store it securely in environment variables.
- 💡Remember to re-run `npx @better-auth/cli@latest generate` after adding or modifying any Better Auth plugins to update your schema correctly.
- 💡Prioritize using environment variables (`BETTER_AUTH_URL`, `BETTER_AUTH_SECRET`) over hardcoding `baseURL` and `secret` in your configuration file for enhanced security and deployment flexibility.
